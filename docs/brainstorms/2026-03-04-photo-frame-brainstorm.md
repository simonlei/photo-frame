# 电子相框应用 Brainstorm

**日期：** 2026-03-04
**作者：** Simon Lei
**状态：** 草稿

---

## 我们在做什么

一个基于老 Android 平板的智能电子相框系统，让家庭成员可以随时将手机上的照片推送到相框展示，让照片从手机相册"活"起来。

核心由两部分组成：
1. **相框端 App**（Android 原生）：运行在平板上，全屏展示照片
2. **上传端**（微信小程序）：家庭成员通过微信小程序选择并上传照片到相框

---

## 为什么选择这个方案

### 选择 Android 原生相框 App

用户明确有 Android pad，选择 Android 原生 App 的好处：
- 可以控制屏幕常亮（防止自动熄屏）
- 可以控制亮度、横竖屏锁定
- 全屏沉浸式展示，无浏览器 UI 干扰
- 可以本地缓存照片，离线也能展示

### 选择微信小程序作为上传端

- 中国用户无需额外安装 App
- 微信生态成熟，照片选择、压缩、上传 API 完善
- 多人绑定同一相框天然支持（通过微信 OpenID 识别用户）

## 关键决策

### 系统架构
```
微信小程序  ──→  后端 API 服务  ──→  照片存储（服务器本地/对象存储）
                      ↓
              Android 相框 App  ──→  展示照片
```

后端部署在自托管服务器上，Android App 通过局域网或公网访问后端。

### 多人共用相框

- 多个微信用户可以"绑定"同一个相框（通过相框 ID 或二维码邀请）
- 所有绑定用户上传的照片都进入同一个相册池
- 可选：标注每张照片的上传者

### 照片展示模式

1. **顺序播放**：按照片拍摄时间（EXIF）或上传时间排序
2. **随机播放**：每次随机选取
3. 每张照片展示时长：用户可配置（默认建议 10-30 秒）
4. 切换动画：淡入淡出、滑动等基础效果

### 照片元信息展示

- 屏幕角落显示：拍摄日期、上传者昵称
- 可开关（相框设置中配置）

### 红色日期提醒（未来功能）

- 系统记录生日、纪念日等特殊日期
- 临近特殊日期时优先展示对应照片或展示提醒信息

## 功能范围（MVP vs 未来）

### MVP（第一版）
- [ ] Android 相框 App：全屏幻灯片，顺序/随机模式，可配置切换时长，多种切换动画
- [ ] 微信小程序：选择本地照片，上传到指定相框
- [ ] 后端 API：照片存储、列表接口、绑定关系管理
- [ ] 多人绑定同一相框（扫码绑定）
- [ ] 相框 App 定时轮询获取新照片（30秒-1分钟）
- [ ] 平板本地设置页（展示时长、播放模式、动画效果）
- [ ] **定时亮屏/黑屏**：支持配置时间段，例如 22:00 黑屏 → 08:00 亮屏
- [ ] App 内自动检查更新并安装
- [ ] 照片元信息显示（日期、上传者昵称，可开关）

### 未来迭代
- 微信公众号机器人（发图片给公众号自动上传）
- 相册分组（按旅行、节日等）
- 红色日期提醒（生日、纪念日临近时优先展示相关照片）
- 照片评论 / 点赞
- 多相框管理

## 技术选型备选

| 组件 | 选项 | 说明 |
|------|------|------|
| 相框 App | Android (Kotlin/Java) | 原生，控制力强 |
| 上传端 | 微信小程序 | 用户无需安装额外 App |
| 后端 | Node.js / Python (FastAPI) | 轻量，自托管友好 |
| 照片存储 | 服务器本地 + Nginx | 简单直接；后期可换对象存储 |
| 数据库 | SQLite / PostgreSQL | 先用 SQLite，够用升 PG |
| 相框-服务器通信 | HTTP 轮询 / WebSocket | 轮询简单，WebSocket 实时性好 |

## 已解决的问题

- **支持多设备？** 先做 Android，后期按需扩展
- **多相框？** 先支持一人一相框，架构上预留多相框扩展
- **公众号？** 后期再加，MVP 不包含
- **部署方式？** 自托管服务器，公网访问，需要域名和 HTTPS
- **照片存储容量？** 暂不限制，先不自动清理旧照片
- **照片压缩策略？** MVP 阶段先不处理，后期再优化
- **APK 分发和更新？** 相框 App 内置自动检查更新机制，自动下载安装新版本
- **相框绑定方式？** 扫码绑定：相框 App 首次启动显示二维码，用户用小程序扫码绑定
- **屏幕方向？** 横竖屏都支持，App 根据平板当前方向自动适配
- **新照片通知？** 相框 App 定时轮询（30秒-1分钟），无需实时推送
- **展示时长？** 用户在平板本地设置页自定义（默认建议 15 秒）
- **配置入口？** 平板本地设置页面，点击屏幕触发
- **切换动画？** 支持多种切换动画（淡入淡出、左右滑动、翻页等），可配置

## 成功标准

- 家人拍了照片，打开小程序，选择照片，上传，1 分钟内在相框上看到这张照片
- 相框 7x24 小时运行，App 不卡死，不被系统强制休眠（通过 `FLAG_KEEP_SCREEN_ON` 保持常亮）
- 定时黑屏：到达用户设定时间后 App 主动将亮度降为 0，外观黑屏但 App 继续运行
- 定时亮屏：到达恢复时间后自动恢复亮度继续展示照片
- 操作简单到老人也能用微信小程序上传照片

> **注意**：常亮（防止系统休眠）和定时黑屏（App 主动调亮度）是两个不同机制，不冲突。
